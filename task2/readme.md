# Rust复杂项目结构说明

## 项目结构

- **rust_complex_project/**
  - **src/**
    - **main.rs**: 主入口文件，负责程序的初始化和控制流。
    - **logger.rs**: 实现线程安全的日志记录器，支持记录和存储日志信息。
    - **string_manager.rs**: 管理字符串的添加和获取功能。
    - **math_utils.rs**: 包含数学工具函数，如阶乘和斐波那契数列。
    - **thread_pool.rs**: 实现简单的线程池，支持并发执行任务。
    - **config.rs**: 定义用于解析命令行参数的配置结构体。
  - **Cargo.toml**: 项目的配置文件，定义基本信息和依赖库。
  - **readme.md**: 项目说明文件。


## 文件和目录说明

### `rust_complex_project/`
这是项目的根目录，包含所有源代码和配置文件。

### `Cargo.toml`
这是Rust项目的配置文件，定义了项目的基本信息和依赖库。它包含以下内容：
- **`name`**: 项目的名称。
- **`version`**: 项目的版本号。
- **`edition`**: Rust语言版本。
- **`dependencies`**: 项目所需的外部库，这里使用了`clap`库来处理命令行参数。

### `src/`
此目录包含所有源代码文件。

#### `main.rs`
主入口文件，负责程序的初始化和控制流。它解析命令行参数，创建日志记录器和字符串管理器，并使用线程池执行任务。

#### `logger.rs`
实现了一个线程安全的日志记录器，支持记录信息并存储所有日志。提供了`log`和`get_logs`方法。

#### `string_manager.rs`
字符串管理器，用于管理字符串的添加和获取功能。实现了添加字符串和获取最长字符串的方法。

#### `math_utils.rs`
包含数学工具函数，例如计算阶乘和斐波那契数列的函数。

#### `thread_pool.rs`
实现了一个简单的线程池，允许将任务分配给多个工作线程并并发执行。支持任务的添加和等待任务完成。

#### `config.rs`
定义了用于解析命令行参数的配置结构体。使用`clap`库来处理输入的线程数量和任务数量。

## 如何运行
在项目根目录中输入：`cargo run -- --threads 4 --tasks 10`
